# Security policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in EpubSharp, please open a **private** report if possible
(for example, via GitHub Security Advisories for this repository), or contact the maintainers directly.

Please include:

- Affected version(s)
- Reproduction steps and/or a proof-of-concept EPUB file
- Impact assessment (what can an attacker do?)

## Supported versions

Security fixes are generally provided for the latest released version and the current development branch.
